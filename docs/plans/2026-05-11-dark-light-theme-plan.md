## Step 2: Wrap layout in `ThemeProvider`

**File:** `app/layout.tsx`

Changes:
- Add `"use client"` is NOT needed here ‚Äî `ThemeProvider` is a client component but can be imported into a server component layout (it's a Provider wrapper).
- Import `ThemeProvider` from `next-themes`.
- Wrap `{children}` with `<ThemeProvider>`.
- Add `suppressHydrationWarning` to `<html>` tag (required by next-themes to prevent hydration mismatch).
- Remove the `darkMode` class on `<html>` since next-themes will manage the `.dark` class via `attribute="class"`.

Full updated file:

```tsx
import type { Metadata } from "next";
import { Geist } from "next/font/google";
import { ThemeProvider } from "next-themes";
import { cn } from "@/lib/utils";
import "./globals.css";

const geist = Geist({ subsets: ["latin"], variable: "--font-sans" });

export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="en" suppressHydrationWarning className={cn("h-full antialiased", "font-sans", geist.variable)}>
      <body className="min-h-full flex flex-col">
        <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
          {children}
        </ThemeProvider>
      </body>
    </html>
  );
}
```

Key details:
- `attribute="class"` ‚Äî adds/removes `.dark` class on `<html>`, matching the existing `.dark { ... }` CSS variables in `globals.css:86`.
- `defaultTheme="system"` ‚Äî respects OS preference on first visit.
- `enableSystem` ‚Äî listens for `prefers-color-scheme` changes.
- `suppressHydrationWarning` on `<html>` prevents next-themes' class injection from triggering a hydration error.

## Step 3: Create theme toggle component

**File:** `components/ui/theme-toggle.tsx` (new)

```tsx
"use client";

import { Moon, Sun } from "lucide-react";
import { useTheme } from "next-themes";
import { Button } from "@/components/ui/button";

export function ThemeToggle() {
  const { theme, setTheme } = useTheme();

  return (
    <Button
      variant="ghost"
      size="icon"
      onClick={() => setTheme(theme === "dark" ? "light" : "dark")}
      aria-label="Toggle theme"
    >
      <Sun className="size-5 rotate-0 scale-100 transition-all dark:-rotate-90 dark:scale-0" />
      <Moon className="absolute size-5 rotate-90 scale-0 transition-all dark:rotate-0 dark:scale-100" />
    </Button>
  );
}
```

Key details:
- `"use client"` ‚Äî required since `useTheme()` is a React hook.
- Uses the existing `Button` component with `variant="ghost"` and `size="icon"`.
- Sun/Moon icons swap via CSS transitions (Sun visible in light mode, Moon in dark).
- `absolute` positioning on Moon so both icons overlay each other.

## Step 4: Create header component

**File:** `components/layout/header.tsx` (new)

```tsx
"use client";

import { ThemeToggle } from "@/components/ui/theme-toggle";

export function Header() {
  return (
    <header className="sticky top-0 z-50 flex h-14 items-center justify-between border-b border-border px-4 bg-background/80 backdrop-blur-sm">
      <span className="text-lg font-semibold text-foreground">Next101</span>
      <ThemeToggle />
    </header>
  );
}
```

Key details:
- Sticky header with backdrop blur for a modern feel.
- Uses `border-border` and `bg-background` from the CSS variable system ‚Äî automatically adapts to dark/light.
- `"use client"` is needed because `ThemeToggle` uses hooks (could alternatively be avoided if we split the toggle out, but this is simpler).

## Step 6: Clean up `app/page.tsx`

**File:** `app/page.tsx`

The current page uses hardcoded `dark:` Tailwind classes (e.g., `dark:bg-black`, `dark:text-zinc-50`). These will conflict with the theme system. Remove the explicit `dark:` variants and rely on the CSS variables from `globals.css` instead.

Changes:
- Replace `bg-zinc-50 dark:bg-black` with `bg-background` (already defined in CSS vars).
- Replace `bg-white dark:bg-black` with `bg-card` or `bg-background`.
- Replace `text-black dark:text-zinc-50` with `text-foreground`.
- Replace `text-zinc-600 dark:text-zinc-400` with `text-muted-foreground`.
- Replace `dark:invert` on images with a conditional approach or remove if not needed.
- Replace hardcoded hover colors with semantic tokens.

## Important notes

- **No changes to `globals.css`** ‚Äî the `.dark { ... }` variables and `@custom-variant dark` are already correctly set up.
- **No changes to `components/ui/button.tsx`** ‚Äî already has dark variant styles.
- **`suppressHydrationWarning`** is critical ‚Äî without it, the initial class injection causes a hydration mismatch error.
- **`Header` is placed in `page.tsx`** (not in `layout.tsx`) to avoid wrapping Server Components in a client boundary. If you later need the header on every page, move it to `layout.tsx` inside the `ThemeProvider` wrapper.
