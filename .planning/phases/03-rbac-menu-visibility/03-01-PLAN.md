<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API (menu-role/batch) | Admin-only endpoint; role extracted from session, not request body |
| client → API (menus/mine) | Any authenticated user; role from session cookie, menus filtered server-side |
| session cookie → role extraction | better-auth verifies token; role string from DB, not client-controlled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-01 | Elevation of Privilege | batchAssignMenus | mitigate | Admin-only middleware checks session.user.role === "admin" before handler |
| T-3-02 | Tampering | getMine role extraction | mitigate | Role read from server-side session via auth.api.getSession(), never from request body |
| T-3-3 | Information Disclosure | getMine response | mitigate | Only returns menus where MenuRole.roleId matches session.user.role; no cross-role leakage |
| T-3-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this phase |
</threat_model>
